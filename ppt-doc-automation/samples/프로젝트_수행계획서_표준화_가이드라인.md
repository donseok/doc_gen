# 프로젝트 수행계획서 표준화 가이드라인

---

## 1. 개요

### 1.1 문서 목적
본 가이드라인은 SI 프로젝트 수행계획서의 표준화 및 자동 문서 생성을 위한 구조와 규칙을 정의합니다.

### 1.2 적용 범위
- SI(System Integration) 프로젝트
- 공공/금융/대기업 IT 프로젝트
- 신규 구축 및 고도화 프로젝트

---

## 2. 표준 템플릿 구조

### 2.1 섹션 구조 (10대 핵심 섹션)

```
┌─────────────────────────────────────────────────────────────┐
│  1. 프로젝트 개요                                              │
│     ├── 프로젝트 정보                                         │
│     ├── 배경 및 필요성                                        │
│     ├── 목적 및 목표                                          │
│     └── 범위 (포함/제외)                                      │
├─────────────────────────────────────────────────────────────┤
│  2. 추진 전략 및 방법론                                        │
│     ├── 핵심 추진 전략                                        │
│     ├── 개발 방법론                                           │
│     └── 기술 아키텍처                                         │
├─────────────────────────────────────────────────────────────┤
│  3. 프로젝트 조직 및 역할                                      │
│     ├── 조직 구성도                                           │
│     ├── R&R (역할과 책임)                                     │
│     └── 투입 인력 현황                                        │
├─────────────────────────────────────────────────────────────┤
│  4. 프로젝트 일정                                              │
│     ├── 마스터 일정                                           │
│     ├── 마일스톤                                              │
│     └── WBS                                                   │
├─────────────────────────────────────────────────────────────┤
│  5. 산출물 관리                                                │
│     ├── 산출물 목록                                           │
│     └── 형상관리 방안                                         │
├─────────────────────────────────────────────────────────────┤
│  6. 품질 관리 계획                                             │
│     ├── 품질 목표/기준                                        │
│     ├── 품질 활동                                             │
│     └── 결함 관리                                             │
├─────────────────────────────────────────────────────────────┤
│  7. 위험 관리 계획                                             │
│     ├── 위험 관리 프로세스                                    │
│     ├── 주요 위험 및 대응                                     │
│     └── 위험 등급 기준                                        │
├─────────────────────────────────────────────────────────────┤
│  8. 의사소통 및 보고 계획                                      │
│     ├── 회의 체계                                             │
│     ├── 보고 체계                                             │
│     └── 소통 채널                                             │
├─────────────────────────────────────────────────────────────┤
│  9. 변경 관리 계획                                             │
│     ├── 변경 프로세스                                         │
│     └── 승인 권한                                             │
├─────────────────────────────────────────────────────────────┤
│ 10. 협조 사항                                                  │
│     ├── 발주기관 협조                                         │
│     └── 연계 시스템 협조                                      │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 변수 표기 규칙

| 표기 방식 | 용도 | 예시 |
|----------|------|------|
| `{{변수명}}` | 단순 텍스트 치환 | `{{프로젝트명}}` |
| `{{변수명: 옵션1/옵션2}}` | 선택형 변수 | `{{보안등급: 대외비/일반}}` |
| `{{목표1}}` ~ `{{목표N}}` | 반복 가능 항목 | 다중 목표 입력 |

### 3.2 변환 파이프라인

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  소스 데이터   │ -> │  표준 템플릿   │ -> │  최종 문서    │
│  (YAML/JSON)  │    │  (Markdown)   │    │ (DOCX/PPTX)  │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
   project_data.yaml  template.md      output.docx
                                       output.pptx
                                       output.pdf
```

### 3.3 PPT 변환 시 슬라이드 매핑

| 섹션 | PPT 슬라이드 | 레이아웃 타입 |
|------|-------------|--------------|
| 1.1 프로젝트 정보 | 슬라이드 1-2 | 표 레이아웃 |
| 1.2~1.4 배경/목적/범위 | 슬라이드 3-5 | 텍스트+다이어그램 |
| 2.x 추진전략/방법론 | 슬라이드 6-8 | 프로세스 다이어그램 |
| 3.x 조직/역할 | 슬라이드 9-10 | 조직도 |
| 4.x 일정 | 슬라이드 11-12 | 간트차트/표 |
| 5.x 산출물 | 슬라이드 13 | 표 레이아웃 |
| 6.x 품질관리 | 슬라이드 14 | 표 레이아웃 |
| 7.x 위험관리 | 슬라이드 15 | 매트릭스 다이어그램 |
| 8.x 의사소통 | 슬라이드 16 | 표 레이아웃 |
| 10.x 협조사항 | 슬라이드 17 | 텍스트+표 |

## 4. 문서 유형별 활용

### 4.1 착수보고서 vs 수행계획서

| 구분 | 착수보고서 | 수행계획서 |
|------|-----------|-----------|
| **목적** | 프로젝트 시작 공식 선언 | 수행 전략/계획 상세 공유 |
| **분량** | 요약본 (15-25 슬라이드) | 상세본 (30-50 페이지) |
| **형식** | PPT 중심 | Word/HWP 중심 |
| **사용 섹션** | 1, 2, 3, 4, 10 위주 | 전체 섹션 |
| **열람 대상** | 경영진, 스폰서 | PM, 설계자, 개발자 |

### 4.2 프로젝트 유형별 커스터마이징

#### 신규 구축 프로젝트
- 2.3 기술 아키텍처 섹션 강화
- 현행 시스템 분석 내용 최소화

#### 고도화/리뉴얼 프로젝트
- 현행 시스템 분석 섹션 추가
- As-Is/To-Be 비교 내용 포함
- 데이터 이관 계획 섹션 추가

#### 유지보수 프로젝트
- SLA 관련 섹션 추가
- 운영 지원 체계 섹션 추가
- 장애 대응 절차 섹션 추가

## 5. 품질 체크리스트

### 5.1 문서 작성 시 점검 사항

| 점검 항목 | 확인 |
|----------|------|
| ☐ 모든 필수 섹션이 포함되었는가? |
| ☐ 모든 변수가 실제 값으로 치환되었는가? |
| ☐ 일정/날짜가 정확하게 기재되었는가? |
| ☐ 조직도와 R&R이 일치하는가? |
| ☐ 산출물 목록이 방법론과 일치하는가? |
| ☐ 마일스톤과 일정이 일치하는가? |
| ☐ 문서 버전과 개정 이력이 업데이트되었는가? |
| ☐ 보안 등급에 맞는 내용만 포함되었는가? |

### 5.2 PPT 변환 시 점검 사항

| 점검 항목 | 확인 |
|----------|------|
| ☐ 슬라이드별 내용량이 적절한가? (1슬라이드 = 1메시지) |
| ☐ 표/다이어그램이 가독성 있게 표시되는가? |
| ☐ 회사 CI/템플릿이 적용되었는가? |
| ☐ 페이지 번호가 정확한가? |
| ☐ 목차와 실제 슬라이드 번호가 일치하는가? |
