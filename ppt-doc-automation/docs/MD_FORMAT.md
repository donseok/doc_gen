# 마크다운(.md) 파일 규격

PPT 자동 생성을 위한 마크다운 파일 작성 규격입니다.

## 프론트매터 (YAML)

파일 상단에 `---`로 감싸진 YAML 메타데이터를 포함합니다.

### 공통 필드
| 필드 | 설명 | 필수 |
|------|------|:----:|
| `type` | 문서 유형 | O |
| `project` | 프로젝트 코드 | O |
| `version` | 문서 버전 | |
| `date` | 작성일 | |
| `author` | 작성자 | |

### 문서 유형 코드
| 코드 | 설명 |
|------|------|
| `project-plan` | 프로젝트 수행계획서 |
| `kickoff-report` | 착수보고서 |
| `weekly-report` | 주간보고서 |
| `monthly-report` | 월간보고서 |
| `interim-report` | 중간보고서 |
| `final-report` | 완료보고서 |

---

## 본문 구조

### 제목 레벨
- `# H1` - 프레젠테이션 전체 제목
- `## H2` - 각 슬라이드의 제목 (슬라이드 구분 기준)
- `### H3` - 슬라이드 내 부제목

### 콘텐츠 요소
- **글머리 기호**: `-` 또는 `*`
- **번호 목록**: `1.`, `2.`, `3.`
- **테이블**: 마크다운 테이블 문법
- **이미지**: `![대체텍스트](이미지경로)`
- **코드 블록**: 백틱 3개로 감싸기

---

## 예시: 주간보고서

```markdown
---
type: weekly-report
project: PRJ-2025-001
week: 3
period_start: 2025-01-13
period_end: 2025-01-17
author: 홍길동
---

# 주간보고서

## 진척 현황
| 구분 | 계획 | 실적 | 차이 |
|------|:----:|:----:|:----:|
| 전체 | 8% | 8% | 0% |

## 금주 실적
- 업무 현황 분석 완료
- 현행 시스템 분석 진행 중

## 차주 계획
- 요구사항 초안 작성

## 이슈 및 위험
| ID | 내용 | 상태 | 담당 |
|----|------|------|------|
| I-001 | 서버 접근 권한 | 진행 | 김철수 |
```

## 예시: 착수보고서

```markdown
---
type: kickoff-report
project: PRJ-2025-001
date: 2025-01-06
author: 홍길동
---

# 프로젝트 착수보고

## 프로젝트 개요
- 프로젝트명: 문서 자동화 시스템
- 기간: 2025.01 ~ 2025.06
- 목표: 업무 효율성 30% 향상

## 추진 전략
1. 단계별 개발 접근
2. 애자일 방법론 적용
3. 주간 진척 보고

## 팀 구성
| 역할 | 이름 | 담당 업무 |
|------|------|----------|
| PM | 홍길동 | 프로젝트 총괄 |
| 개발 | 김철수 | 백엔드 개발 |
| 개발 | 이영희 | 프론트엔드 개발 |

## 주요 일정
| 단계 | 기간 | 산출물 |
|------|------|--------|
| 분석 | 1~2주차 | 요구사항 정의서 |
| 설계 | 3~4주차 | 시스템 설계서 |
| 개발 | 5~20주차 | 소스코드 |
| 테스트 | 21~24주차 | 테스트 결과서 |
```

## 파싱 규칙 요약

1. **슬라이드 분할**: `## H2` 헤딩을 기준으로 슬라이드 분할
2. **제목 추출**: 첫 번째 `# H1`이 프레젠테이션 제목
3. **메타데이터**: YAML 프론트매터에서 문서 정보 추출
4. **레이아웃 자동 선택**:
   - 이미지만 있는 경우: 이미지 전용 레이아웃
   - 이미지 + 텍스트: 이미지-콘텐츠 레이아웃
   - 코드 블록: 코드 레이아웃
   - 기본: 제목-콘텐츠 레이아웃
